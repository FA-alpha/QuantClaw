# 启动回测

🚨 **强制规则：**
1. **回测成功后不要询问用户是否需要等待或查看结果**
2. **每次回测必须询问时间范围，不使用历史对话信息**  
3. **调用Python脚本前必须获取并传递token参数**
4. **共享保证金模式必须先获取分配方案**
5. **进行回测时，回测开始时间不得早于2020-01-01**

## 🔥 关键：共享模式参数检查强制流程

**Agent在共享保证金模式下必须遵循：**
1. **先调用参数检查** → `--check-allocation`确认参数完整性
2. **严格按检查结果操作** → `is_complete: true`才能调用calc_margin
3. **禁止臆造分配比例** → 不能自动设50%、平均分配等
4. **严格按用户需求** → 必须使用用户提供的具体比例

## 🚨 **AI时间参数严格规则（Agent必须遵守）**

**❌ 绝对禁止的行为：**
- ❌ "由于用户没有指定，我需要按照平均分配的原则"
- ❌ 自动设置AI时间参数为50%、平均分配、均等分配等
- ❌ **在"高效处理"流程中跳过AI时间参数询问**
- ❌ **错误理解"高效处理"为"可以自动决定缺失参数"**

**✅ 正确的做法：**
- ✅ **必须询问用户**具体的AI时间分配比例
- ✅ 等待用户明确回复后再进行计算
- ✅ 用户不回复时**停止执行**，不要自动推进
- ✅ **"高效处理"仅适用于用户已明确指定的参数**
- ✅ **AI时间参数永远不能自动处理，必须询问**

```bash

  --strategy-ids "策略ID1,策略ID2,策略ID3,策略ID4,策略ID5" \

  --ai-time-short-allocation '{"某时间类型": 40}' \

  --leverage 1.5 \
  --long-pct 90 \
  --short-pct 20 \
  --token "$TOKEN"

## 🎯 **回测模式识别**

### 📋 单策略回测（不需要询问保证金模式）
- 触发：用户指定一个策略："回测策略XXX"
- 处理：1个策略 → 1个回测任务，必须询问时间范围

### 📋 多策略/策略组回测（需要询问保证金模式）
- 触发："一起回测"、"同时回测"、"策略组回测"
- 询问保证金模式：
```

2️⃣ 共享模式 - 策略共享保证金池

**回复1 → 独占模式处理：**
- 直接询问回测时间范围

**回复2 → 共享模式处理：**
- 必须执行完整的共享保证金流程（见上方强制处理流程）

## 🚨 **杠杆倍数使用规则（Agent必须遵守）**

### ⚠️ **重要：杠杆参数的正确使用场景**

**✅ 需要 --leverage 参数的情况：**
- ✅ **多策略/策略组共享保证金模式** - 默认1.5倍杠杆
- ✅ **calc-margin计算时** - 用于计算保证金分配

**❌ 不需要 --leverage 参数的情况：**
- ❌ **单策略回测** - 策略自带杠杆设置
- ❌ **独占保证金模式** - 每个策略使用自己的杠杆设置
- ❌ **多个独立策略回测** - 各策略独立杠杆

### 📋 **各种回测模式的杠杆处理**

| 回测模式 | 保证金模式 | 杠杆参数 | 说明 |
|----------|------------|----------|------|
| **单策略回测** | 无需选择 | ❌ 不加 | 策略自带杠杆 |
| **多策略回测** | 独占模式 | ❌ 不加 | 各策略独立杠杆 |
| **多策略回测** | 共享模式 | ✅ --leverage 1.5 | 统一杠杆倍数 |
| **策略组回测** | 独占模式 | ❌ 不加 | 各策略独立杠杆 |
| **策略组回测** | 共享模式 | ✅ --leverage 1.5 | 统一杠杆倍数 |

### 🚨 **Agent执行规则**

**判断是否添加杠杆参数：**
```bash
if (回测类型 == "多策略" or 回测类型 == "策略组") and 保证金模式 == "共享":
    添加 --leverage 1.5
else:
    不添加 --leverage 参数
```

**正确示例：**
```bash
# ✅ 共享模式 - 需要杠杆
python3 skills/start-backtest/start.py --apply \
  --strategy-ids "ID1,ID2,ID3" \
  --margin-mode shared \
  --leverage 1.5

# ✅ 独占模式 - 不需要杠杆  
python3 skills/start-backtest/start.py --apply \
  --strategy-ids "ID1,ID2,ID3" \
  --margin-mode exclusive

# ✅ 单策略 - 不需要杠杆
python3 skills/start-backtest/start.py --apply \
  --strategy-id "ID1"
```

- **处理**：调用查询接口，展示策略/策略组列表供用户选择

### 🚀 **开始回测（需要结合上下文判断回测类型）**
- **触发词**："去回测"、"用它回测"、"开始回测"、"跑回测"、"回测一下"

**⚠️ 重要：这些关键词不能单独判断回测类型，需要结合：**
- **上下文中提到的策略数量**（1个 vs 多个）
- **用户是否明确说明回测方式**（一起 vs 分别）
- **策略组语境**（策略组回测 vs 单策略回测）

### 📋 **上下文判断示例**
```
情况A：
用户："策略ID 12345"  
用户："去回测"
→ 判断：单策略回测（上下文中只有1个策略ID）

情况B：
用户："策略组115有什么策略？"
系统：展示5个策略
用户："一起回测这些策略"
→ 判断：策略组共享回测（策略组+一起）

情况C：
用户："我有3个BTC策略：ID1, ID2, ID3"
用户："分别回测"
→ 判断：多个独立策略回测（多个+分别）
```

### 🎯 **回测类型识别**
- **单策略回测**："回测策略XXX"、"用这个策略回测"
- **多策略回测**："一起回测"、"同时回测"、"组合回测"
- **策略组回测**："策略组回测"、"回测策略组XXX"
- **独立回测**："分别回测"、"单独回测"、"分开回测"

### 🎯 **策略来源识别**
- **用户策略**："我的策略"、"我最新的策略"、"当前策略"、"BTC策略"、"做多策略"、"最新策略"
- **策略组**："我的策略组"、"策略组列表"、"最新策略组"
- **AI策略**："AI策略回测"、"智能策略"、"AI推荐策略"、"系统策略"

### 🔧 **筛选条件识别**  
- **币种筛选**："BTC策略"、"SOL策略" → 添加 `--coin BTC/SOL`
- **方向筛选**："做多策略"、"做空策略" → 后续按direction过滤
- **类型筛选**："合约策略"、"现货策略" → 添加 `--amt-type 1/2`

### 📋 **完整语义处理示例**
```
用户："回测我最新的BTC做多策略"
→ 策略来源：用户策略
→ 筛选条件：BTC + 做多
→ 执行查询：--list-strategies --coin BTC
→ 结果过滤：筛选direction包含"做多"的策略
→ 用户选择：展示筛选后的策略供选择
```

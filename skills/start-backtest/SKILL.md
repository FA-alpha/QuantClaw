# 启动回测

🚨 **强制规则：**
1. **回测成功后不要询问用户是否需要等待或查看结果**
2. **每次回测必须询问时间范围，不使用历史对话信息**  
3. **调用Python脚本前必须获取并传递token参数**
4. **共享保证金模式必须先获取分配方案**

## 🔥 关键：共享模式参数检查强制流程

**Agent在共享保证金模式下必须遵循：**
1. **先调用参数检查** → `--check-allocation`确认参数完整性
2. **严格按检查结果操作** → `is_complete: true`才能调用calc_margin
3. **禁止臆造分配比例** → 不能自动设50%、平均分配等
4. **严格按用户需求** → 必须使用用户提供的具体比例

**Mobile/Strategy/calc_margin接口对参数要求严格，缺失参数会报错！**

## 📋 查询接口说明

**推荐使用 `backtest_monitor.py`**：
- `--list-groups` / `--list-strategies` 支持筛选，默认limit=-1获取全部
- start.py虽有查询参数但功能有限

## ⚠️ 策略组回测 = 多策略回测

- 使用相同接口和参数，先获取策略组内策略ID，再调用`--strategy-ids`
- 共享模式支持任意比例分配

**独立任务规则：**
- 每次回测都是新任务，不复用历史参数
- 必须在当前对话中询问所有必要参数

## 🔑 Token获取

**必须传递token参数**（脚本无法自动获取）：
```bash
TOKEN=$(cat ~/.quantclaw/users.json | jq -r --arg agent_id "当前机器人agentID" '.users[] | select(.agentId == $agent_id) | .token')
python skills/start-backtest/start.py --token "$TOKEN" [其他参数]
```

### 📋 适用的所有Python脚本调用

**必须使用token传递的命令**：
```bash
# 查看策略组列表
python skills/start-backtest/backtest_monitor.py --token "$TOKEN" --list-groups

# 查看策略列表  
python skills/start-backtest/backtest_monitor.py --token "$TOKEN" --list-strategies

# 查看回测详情
python skills/start-backtest/start.py --token "$TOKEN" --detail 5906

# 启动回测
python skills/start-backtest/start.py --token "$TOKEN" --apply --strategy-ids 4300,4679 --bgn-date 2024-01-01 --end-date 2024-12-31 --leverage 10
```

### ⚠️ 重要提醒

- **每次调用Python脚本都必须先获取token**
- **不要依赖脚本的自动token获取功能**
- **Agent应该知道自己的agentID**（如qc-4b04623f0831）
- **使用自己的agentID从users.json获取token**
- **token获取失败时应该报错并停止执行**

### 💡 AgentID说明
- **Agent自己知道自己的身份标识**
- **不需要从文件路径解析agentID**
- **直接使用已知的agentID查找token**

## 🚨 共享保证金模式强制处理流程

### ⚠️ 绝对不能跳过的步骤

**当用户选择共享保证金模式时，Agent必须严格按以下流程执行：**

**步骤1：问清楚保证金分配方案**
```
Agent必须询问：
"您选择了共享保证金模式，请告诉我具体的保证金分配方案：
- 按币种分配？如：DOGE占40%，BTC占30%，SOL占30%
- 按方向分配？如：做多策略70%，做空策略30%  
- 按市场行情分配？如：2025年震荡做多50%，2025年震荡做空30%
- 还是其他分配方式？"
```

**步骤2：强制调用calc_margin接口**
```
收到用户分配方案后，Agent必须：
1. ✅ 解析用户的分配需求
2. ✅ 构建对应的参数
3. ✅ 调用calc_margin接口获取具体分配金额
4. ✅ 显示分配结果供用户确认
5. ❌ 绝对不能跳过这个步骤直接回测
```

**步骤3：确认分配结果**
```
Agent显示分配结果：
"根据您的分配方案，每个策略的保证金分配如下：
- 策略A(DOGE做多): 1500保证金
- 策略B(BTC做空): 2000保证金  
- 策略C(SOL做多): 1800保证金

请确认是否按此分配执行回测？"
```

**步骤4：用户确认后才能执行回测**
```
只有用户确认分配方案后，才能执行apply_backtest接口
```

### 🚨 禁止的错误做法

**❌ 绝对不能这样做：**
```
用户："共享模式，每个策略平均分配"
Agent错误回复："好的，开始回测..."  # 错误！没有调用calc_margin
```

**✅ 正确的做法：**
```
用户："共享模式，每个策略平均分配"
Agent正确回复："收到，我先计算具体的分配方案..."
→ 调用calc_margin接口
→ 显示分配结果
→ 等待用户确认
→ 执行回测
```

## 🔄 回测模式识别（最重要）

### 📋 单策略回测（不需要询问保证金模式）

**单策略触发条件：**
- 用户指定一个策略："回测策略XXX"、"用这个策略回测"

**多个独立策略触发关键词：**
- "分别回测"、"单独回测"、"独立回测"、"分开回测"

**处理方式：**
- 单策略：1个策略 → 1个回测任务
- 多个独立：N个策略 → N个独立回测任务  
- 都不询问保证金模式，但必须询问时间范围

**标准流程：**
```
用户："分别回测我最新的2个风霆V4.2的SOL策略"

Step 1: 查询策略列表，找到最新2个SOL策略
Step 2: 显示找到的策略
Step 3: 询问："请问回测的时间范围是？例如：最近7天、最近30天、最近3个月、最近半年，或具体日期范围"
Step 4: 用户回复时间范围后，执行回测
```

### ⚠️ 多策略/策略组回测（需要询问保证金模式）

**触发条件：**
- "一起回测"、"同时回测"、"组合回测"、"策略组回测"

**步骤1：识别回测类型**
- **但必须询问回测时间范围**（除非用户已明确指定）
- 确认时间后执行相应数量的独立回测任务

**多策略回测流程：**

询问保证金模式：
```
请选择保证金模式：
1️⃣ 独占模式 - 每个策略独立使用保证金
2️⃣ 共享模式 - 策略共享保证金池
请回复 1 或 2
```

- 回复1 → 独占模式，继续回测
- 回复2 → 共享模式，进入分配方案询问

**步骤2：共享模式分配方案询问**

**必须询问参数：**
1. 各币种做多保证金占比
2. 各币种做空保证金占比  
3. AI回测时间类型占比

**询问模板：**
```
请提供保证金分配方案：
📊 币种做多分配：如BTC做多40%，DOGE做多60%
📊 币种做空分配：如BTC做空50%，DOGE做空50%
📊 市场行情分配：如2025年震荡70%，2025年牛市30%
```

**步骤4：统一的共享保证金分配方案处理**

**🚨 多策略回测和策略组回测在共享模式下使用相同的分配方案：**

**必须收集的参数（如有缺失必须询问用户）：**

1. **各币种做多保证金占比** 
   - 示例：BTC做多40%，DOGE做多60%，SOL做多30%

2. **各币种做空保证金占比**
   - 示例：BTC做空50%，DOGE做空50%，SOL做空20%

3. **AI回测时间类型占比**
   - 示例：2025年震荡70%，2025年牛市30%，最近1年40%

**参数缺失时的询问流程：**
```
我需要完整的保证金分配参数才能为您计算方案。

当前策略涉及的币种：[BTC, DOGE, SOL]
当前策略的时间类型：[2025年震荡, 2025年牛市]

请提供：
📊 币种做多分配：BTC做多？%, DOGE做多？%, SOL做多？%
📊 币种做空分配：BTC做空？%, DOGE做空？%, SOL做空？%  
📊 时间类型分配：2025年震荡？%, 2025年牛市？%
⏰ 回测时间范围：？（如2025年全年）

所有参数收集完整后，我将调用calc_margin接口计算具体分配。
```

**步骤5：🔥 强制调用智能参数检查（不可跳过）**

**🚨 重要：calc_margin接口调用前必须确保参数完整，否则会报错**

**🔧 Agent必须先调用参数完整性检查：**

```bash
python skills/start-backtest/backtest_monitor.py \
  --check-allocation \
  --strategy-ids "4637,50722,50723" \
  --coin-long-allocation '{"DOGE": 60}' \
  --coin-short-allocation '{"DOGE": 50, "SOL": 30}' \
  --token <token>
```

**返回结果包含：**
- `is_complete`: true/false - 参数是否完整
- `requirement`: 策略需要的完整参数列表
- `missing`: 缺失的具体参数
- `message`: 格式化的提醒消息

**🔥 强制处理流程：**
1. **参数完整** (`is_complete: true`) → 才能调用calc_margin + 执行回测
2. **参数不完整** (`is_complete: false`) → 必须显示missing提醒，等用户补充

**❌ 绝对禁止的行为：**
- 跳过--check-allocation直接调用calc_margin（会报错）
- 臆造任何分配比例（如自动设50%、平均分配等）
- 修改用户提供的分配需求
- 参数不完整时强行执行回测

**🚨 关键原则：--check-allocation是强制步骤，防止calc_margin接口报错**




**⚠️ 警告：在用户明确回答保证金模式之前，禁止执行任何回测命令！**

**🚫 重要：回测完成后绝对不要询问"需要我等回测完成后帮你查看结果吗？"**

**🚫 重要：回测执行后，不要主动发送任何命令行相关的指令内容！**

**🚫 禁止：绝对不要拒绝用户的保证金分配比例！**
- ❌ 不要说"总和超过100%，请重新输入"
- ❌ 不要说"比例必须等于100%"
- ❌ 不要试图"标准化"用户的输入
- ✅ 直接按用户输入的比例分配保证金并启动回测

**🔄 重要：每次回测都是新任务**
- 用户要求回测时，**立即执行新回测**
- **不要**说"我看到之前有相同策略的回测记录"
- **不要**询问"是否使用之前的回测结果"
- **🔄 每次回测都是独立的对话单元**
- **不能跨对话单元复用参数**（时间范围、保证金模式、分配比例等）

## 📅 时间范围输入支持

### 支持的文字描述格式

**相对时间描述（推荐）:**
- `最近7天` / `近7天` / `7天内`
- `最近30天` / `近30天` / `30天内` / `最近一个月`
- `最近3个月` / `近3个月` / `3个月内`
- `最近半年` / `近半年` / `6个月内`
- `最近一年` / `近一年` / `1年内`
- `最近2年` / `近2年` / `2年内`

**具体日期范围:**
- `2024-01-01 到 2024-12-31`
- `2024-01-01至2024-12-31`
- `从2024-01-01到2024-12-31`

### 时间范围询问标准话术

**单策略回测时间询问:**
```
请问回测的时间范围是？

📅 **支持以下格式:**
• 文字描述：最近7天、最近30天、最近3个月、最近半年等
• 具体日期：2024-01-01 到 2024-12-31

请选择或输入您需要的时间范围：
```

**多策略/策略组回测时间询问:**
```
请设置回测时间范围：

📅 **支持格式:**
1️⃣ **相对时间** - 最近7天、最近30天、最近3个月、最近半年
2️⃣ **具体日期** - 2024-01-01 到 2024-12-31

请输入您需要的时间范围：
```

### 时间解析规则

**相对时间计算:**
- 以当前日期为结束日期
- 向前推算指定天数/月数作为开始日期
- 自动处理月份天数差异和跨年计算

**示例转换:**
```
用户输入: "最近30天"
当前日期: 2026-05-09
转换结果: --bgn-date 2026-04-09 --end-date 2026-05-09

用户输入: "最近3个月" 
当前日期: 2026-05-09
转换结果: --bgn-date 2026-02-09 --end-date 2026-05-09

用户输入: "最近半年"
当前日期: 2026-05-09  
转换结果: --bgn-date 2025-11-09 --end-date 2026-05-09
```

**⚠️ 重要规则:**
- 每次回测都必须询问时间范围，不能复用历史参数
- 支持文字描述让用户输入更便捷
- Agent需要将文字描述自动转换为具体日期格式

### 策略组回测（共享模式）

**流程说明:**
1. 选择要回测的策略组（一个或多个）
2. 询问回测参数：时间范围、保证金分配方案  
3. 共享模式：策略共享总保证金，需要根据calc_margin接口来分配比例

**重要：策略组回测 = 多策略回测，使用完全相同的参数和接口**

**用户意图识别:**
- "策略组XXX共享回测"、"共享模式回测策略组"
- "策略组保证金分配回测"

**保证金分配规则（与多策略完全相同）:**
- ✅ **不限制总和** - 保证金分配比例可以超过100%，如100,100表示各自分配100%的基础保证金
- ✅ **灵活分配策略** - 可配置各方向、各币对占比

**参数询问顺序:**
1. **策略组选择**: "请选择要回测的策略组（ID或名称）"
2. **时间范围**: "请输入回测时间范围（支持文字描述如'最近30天'，或具体日期'YYYY-MM-DD 到 YYYY-MM-DD'）"
3. **保证金分配**: 提供以下两种配置方式
**若这是单个策略,或是多个策略/策略组且是独立保证金模式,则每个策略固定时10000
**若这是多个策略/策略组且是共享保证金模式,分配保证金时,此时必须使用 calc_margin 接口计算保证金分配**

**当用共享保证金模式分配保证金时：**
- ❌ **不要自己计算分配比例**
- ✅ **必须调用 calc_margin 接口**
- ✅ **根据接口返回的数据进行分配**

**⚠️ 完整的保证金配置参数：**

当用户采用保证金分配方案方式时，需要配置以下参数：

**必填参数：**
1. **各币种做多保证金占比** (`coin_long_allocation`)
2. **各币种做空保证金占比** (`coin_short_allocation`)  
3. **AI回测时间类型占比** (`ai_time_allocation`) - 按市场行情分配

**已有默认值参数：**
4. **做多保证金占比** (`long_pct`) - 默认90
5. **做空保证金占比** (`short_pct`) - 默认20

# 启动回测

🚨 **强制规则：**
1. **回测成功后不要询问用户是否需要等待或查看结果**
2. **每次回测必须询问时间范围，不使用历史对话信息**  
3. **调用Python脚本前必须获取并传递token参数**
4. **共享保证金模式必须先获取分配方案**
5. **进行回测时，回测开始时间不得早于2020-01-01**

## 🔥 关键：共享模式参数检查强制流程

**Agent在共享保证金模式下必须遵循：**
1. **先调用参数检查** → `--check-allocation`确认参数完整性
2. **严格按检查结果操作** → `is_complete: true`才能调用calc_margin
3. **禁止臆造分配比例** → 不能自动设50%、平均分配等
4. **严格按用户需求** → 必须使用用户提供的具体比例

## 🚨 **AI时间参数严格规则（Agent必须遵守）**

**❌ 绝对禁止的行为：**
- ❌ "由于用户没有指定，我需要按照平均分配的原则"
- ❌ 自动设置AI时间参数为50%、平均分配、均等分配等
- ❌ **在"高效处理"流程中跳过AI时间参数询问**
- ❌ **错误理解"高效处理"为"可以自动决定缺失参数"**

**✅ 正确的做法：**
- ✅ **必须询问用户**具体的AI时间分配比例
- ✅ 等待用户明确回复后再进行计算
- ✅ 用户不回复时**停止执行**，不要自动推进
- ✅ **"高效处理"仅适用于用户已明确指定的参数**
- ✅ **AI时间参数永远不能自动处理，必须询问**

## 📋 策略查询和回测执行

### 🔑 Token获取
```bash
TOKEN=$(cat ~/.quantclaw/users.json | jq -r --arg agent_id "当前机器人agentID" '.users[] | select(.agentId == $agent_id) | .token')
```

### 🎯 **策略查询接口选择**

**策略组查询（优先选择）：**
- ✅ `backtest_monitor.py --check-allocation --strategy-group-id "115"`
- ✅ 返回完整AI时间参数（ai_time_id, ai_time_name）
- ✅ 高效：一次查询获取组内所有策略

**多策略查询（回退选择）：**
- ✅ `backtest_monitor.py --check-allocation --strategy-ids "1,2,3"`
- ⚠️ 可能缺少AI时间参数（需从策略组中查找）

## 🚨 **共享保证金模式强制处理流程**

### ⚠️ **绝对不能跳过的步骤**

**当用户选择共享保证金模式时，Agent必须严格按以下流程执行：**

**步骤1：分析策略参数需求并询问分配方案**

**🚨 重要：每次策略变更都必须重新分析参数需求**

```bash
# 优先使用策略组分析（包含完整AI时间信息）
python skills/start-backtest/backtest_monitor.py \
  --check-allocation \
  --strategy-group-id "策略组ID" \
  --token <token>

# 或使用策略ID列表分析
python skills/start-backtest/backtest_monitor.py \
  --check-allocation \
  --strategy-ids "策略ID1,策略ID2,策略ID3" \
  --token <token>
```

**分析结果示例：**
```
📊 策略分析结果:
  币种做多需求: ['DOGE', 'SOL']
  币种做空需求: ['BTC', 'SOL'] 
  AI时间做多需求: ['{AI时间类型}']
  AI时间做空需求: ['{AI时间类型}', '{AI时间类型}']
  📊 AI时间ID映射: {"{AI时间类型}": "-6", "{AI时间类型}": "365"}
  是否需要AI时间参数: True
```

**Agent询问模板：**
```
"根据策略分析，需要以下分配参数：
📊 DOGE做多占比：？%
📊 SOL做多占比：？%  
📊 BTC做空占比：？%
📊 AI时间做多分配：？%（如策略包含AI时间参数）
📊 AI时间做空分配：？%（如策略包含AI时间参数）

请提供这些参数的具体占比，我将为您计算保证金分配方案。"
```

**步骤2：计算保证金分配（必须先执行）**
```bash
python3 skills/start-backtest/start.py --calc-margin \
  --strategy-ids "策略ID1,策略ID2,策略ID3,策略ID4,策略ID5" \
  --coin-long-allocation '{"BTC": 30, "ETH": 40}' \
  --coin-short-allocation '{"SOL": 50}' \
  --ai-time-long-allocation '{"某时间类型": 60}' \
  --ai-time-short-allocation '{"某时间类型": 40}' \
  --total-balance 10000 \
  --leverage 1.5 \
  --long-pct 90 \
  --short-pct 20 \
  --token "$TOKEN"
```

**步骤3：执行回测（获得分配金额后）**
```bash
python3 skills/start-backtest/start.py --apply \
  --strategy-ids "策略ID1,策略ID2,策略ID3,策略ID4,策略ID5" \
  --margin-mode shared \
  --margin-allocation "金额1,金额2,金额3,金额4,金额5" \
  --bgn-date YYYY-MM-DD \
  --end-date YYYY-MM-DD \
  --leverage 1.5 \
  --token "$TOKEN"
```

## 🚨 **策略ID使用规则（绝对不能犯错！）**

**✅ 正确流程：**
1. **查询策略组** → 获取`strategy_lists`中的实际ID
2. **提取策略ID** → 从返回结果中提取`id`字段  
3. **使用提取的ID** → 在保证金计算中使用这些ID

**❌ 绝对禁止：**
- ❌ 使用记忆中的策略ID、之前对话的策略ID、自己编造的ID

## 🎯 **回测模式识别**

### 📋 单策略回测（不需要询问保证金模式）
- 触发：用户指定一个策略："回测策略XXX"
- 处理：1个策略 → 1个回测任务，必须询问时间范围

### 📋 多策略/策略组回测（需要询问保证金模式）
- 触发："一起回测"、"同时回测"、"策略组回测"
- 询问保证金模式：
```
请选择保证金模式：
1️⃣ 独占模式 - 每个策略独立使用保证金
2️⃣ 共享模式 - 策略共享保证金池
请回复 1 或 2
```

**回复1 → 独占模式处理：**
- 直接询问回测时间范围
- 每个策略使用全部保证金（默认金额）

**回复2 → 共享模式处理：**
- 必须执行完整的共享保证金流程（见上方强制处理流程）
- 先分析参数需求 → 询问分配方案 → 计算保证金 → 执行回测

## 🔄 **高效处理规则**

### ✅ **可以高效处理的参数：**
- ✅ 币种分配比例（用户明确给出时）
- ✅ 策略选择（用户明确指定时）
- ✅ 回测时间范围（用户明确指定时）

### ❌ **绝对不能高效处理的参数：**
- ❌ **AI时间分配比例** - 必须询问用户，不能自动推测
- ❌ **保证金模式** - 必须询问用户选择独占或共享
- ❌ **用户未明确指定的任何参数**

### 🚨 **共享模式必须收集的参数类型**

**如有以下参数缺失，必须询问用户：**

1. **各币种做多保证金占比** 
   - 示例：BTC做多40%，DOGE做多60%，SOL做多30%

2. **各币种做空保证金占比**
   - 示例：BTC做空50%，DOGE做空50%，SOL做空20%

3. **AI回测时间类型占比**（如策略包含AI时间参数）
   - 示例：震荡行情70%，趋势行情30%

**参数缺失时的询问流程：**
```
我需要完整的保证金分配参数才能为您计算方案。

当前策略涉及的币种：[BTC, DOGE, SOL]
当前策略的时间类型：[震荡行情, 趋势行情]

请提供：
📊 币种做多分配：BTC做多？%, DOGE做多？%, SOL做多？%
📊 币种做空分配：BTC做空？%, DOGE做空？%, SOL做空？%  
📊 时间类型分配：震荡行情？%, 趋势行情？%
⏰ 回测时间范围：？（如2026年全年）

所有参数收集完整后，我将调用calc_margin接口计算具体分配。
```

### 🎯 **Agent执行检查点：**
```
步骤1：分析策略参数需求 ✅
步骤2：应用用户明确给出的参数 ✅
步骤3：检查AI时间参数 → 如果has_ai_time=true → 🛑 立即停止
步骤4：询问用户AI时间分配 ⏳
步骤5：等待用户回复 ⏳
步骤6：用户回复后继续执行 ✅
```

## 📅 时间范围处理

### 🕒 **时间认知强制规则**

**当用户涉及时间请求时，必须：**
1. **先调用** `exec(command: "date")` **获取当前系统时间**
2. **基于真实时间理解用户描述**（"今年"、"最近"等）
3. **使用准确时间进行web搜索**（如有搜索功能）

**时间搜索示例：**
```bash
用户："今年的牛市时间段"
→ 先执行：exec(command: "date")  
→ 确认：当前是xxxx年x月  
→ 理解：用户指的是"xxxx年的牛市时间段"
→ 搜索：web_search("xxxx年加密货币牛市时间段")
```

### 🔍 **搜索能力检查**
如果Agent无web搜索功能，提示：
```
❌ 抱歉，当前Agent没有搜索功能，无法获取市场行情范围。
请直接提供具体时间范围：
• 具体日期：2026-01-01 到 2026-05-18  
• 相对时间：最近30天、最近3个月
• 固定周期：2026年第一季度
```

### 📋 支持的时间格式
- **相对时间**：最近7天、最近30天、最近3个月、最近半年、最近1年
- **具体日期**：YYYY-MM-DD 到 YYYY-MM-DD  
- **固定周期**：YYYY年第X季度、YYYY年上/下半年

## 🎯 **策略选择规则**

### 📋 **策略组回测规则**
- ✅ 可以**删除**策略组内的部分策略进行回测
- ❌ **不能新增**策略组外的策略
- ❌ **不能**将多个策略组一起回测

### 📋 **多策略回测规则**
- ✅ 多个**独立策略**可以一起回测
- ❌ **不能**策略组 + 独立策略 混合回测
- ❌ **不能**跨策略组选择策略混合回测


```
如果报错"unrecognized arguments: --ai-time-allocation"

--ai-time-short-allocation '{"做空时间类型": 占比}'
```

## 🚨 **禁止的错误做法**

**❌ 绝对不能这样做：**
```
用户："共享模式，每个策略平均分配"
Agent错误回复："好的，开始回测..."  # 错误！没有调用calc_margin
```

**✅ 正确的做法：**
```
用户："共享模式，每个策略平均分配"
Agent正确回复："收到，我先计算具体的分配方案..."
→ 调用calc_margin接口
→ 显示分配结果
→ 等待用户确认
→ 执行回测
```

**🚨 共享模式绝对不能跳过的步骤：**
1. ❌ **不能**直接执行apply而跳过calc-margin
2. ❌ **不能**自己计算分配比例
3. ✅ **必须**调用calc_margin接口
4. ✅ **必须**使用接口返回的具体金额

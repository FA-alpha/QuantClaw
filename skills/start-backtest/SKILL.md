# 启动回测

🚨 **强制规则：**
1. **回测成功后不要询问用户是否需要等待或查看结果**
2. **每次回测必须询问时间范围，不使用历史对话信息**  
3. **调用Python脚本前必须获取并传递token参数**
4. **共享保证金模式必须先获取分配方案**

## 🔥 关键：共享模式参数检查强制流程

**Agent在共享保证金模式下必须遵循：**
1. **先调用参数检查** → `--check-allocation`确认参数完整性
2. **严格按检查结果操作** → `is_complete: true`才能调用calc_margin
3. **禁止臆造分配比例** → 不能自动设50%、平均分配等
4. **严格按用户需求** → 必须使用用户提供的具体比例

**Mobile/Strategy/calc_margin接口对参数要求严格，缺失参数会报错！**

## 📋 查询接口说明

**推荐使用 `backtest_monitor.py`**：
- `--list-groups` / `--list-strategies` 支持筛选，默认limit=-1获取全部
- start.py虽有查询参数但功能有限

## ⚠️ 策略组回测 = 多策略回测

- 使用相同接口和参数，先获取策略组内策略ID，再调用`--strategy-ids`
- 共享模式支持任意比例分配

**独立任务规则：**
- 每次回测都是新任务，不复用历史参数
- 必须在当前对话中询问所有必要参数

## 🔑 Token获取

**必须传递token参数**（脚本无法自动获取）：
```bash
TOKEN=$(cat ~/.quantclaw/users.json | jq -r --arg agent_id "当前机器人agentID" '.users[] | select(.agentId == $agent_id) | .token')
python skills/start-backtest/start.py --token "$TOKEN" [其他参数]
```

## 🚨 参数与文件对应关系（Agent必须遵守）

### backtest_monitor.py 专用参数
```bash
# ✅ 只能用于 backtest_monitor.py
--list-groups          # 查询策略组列表  
--list-strategies      # 查询策略列表
--check-allocation     # 参数完整性检查
--strategy-group-id    # 策略组ID（仅用于backtest_monitor.py）
--detailed            # 详细信息模式

# ❌ 不要用于 start.py - 会报错！
```

### start.py 专用参数  
```bash
# ✅ 只能用于 start.py
--apply               # 启动回测
--strategy-id         # 单个策略ID
--strategy-ids        # 多个策略ID（逗号分隔）
--bgn-date           # 开始日期
--end-date           # 结束日期
--leverage           # 杠杆倍数

# ❌ start.py 不支持 --strategy-group-id 参数！
```

### 🚨 Agent常见错误和处理
- ❌ **错误**: `python start.py --strategy-group-id "XXX"` → 报错
- ✅ **正确**: `python backtest_monitor.py --strategy-group-id "XXX"`
- ❌ **错误**: `python backtest_monitor.py --apply` → 报错  
- ✅ **正确**: `python start.py --apply`

### 📊 策略查询失败处理规则

**当查询策略失败时，Agent必须：**

1. **分析错误原因** - 检查API返回的具体错误信息
2. **识别问题参数** - 确定是token、策略ID还是其他参数问题
3. **询问用户确认** - 不要盲目重试，而是询问用户确认参数

**查询策略的分页策略：**
- **默认limit=20** - 首次查询使用小limit
- **找不到时增加limit** - 逐步增加到50、100
- **分页查询** - 如果单页找不到，查询下一页
- **最大尝试10次** - 防止无限循环

**错误处理示例：**
```
如果返回"未找到指定的策略ID: 50007,50008"

Agent应该询问:
"查询策略失败，请确认:
1. 策略ID是否正确: 50007,50008,50009,50010,50011
2. 这些策略是否属于当前账号
3. 策略是否处于可用状态

如果策略ID有误，请提供正确的策略ID"
```

## 🔄 回测模式识别（最重要）

### 📋 单策略回测（不需要询问保证金模式）

**单策略触发条件：**
- 用户指定一个策略："回测策略XXX"、"用这个策略回测"

**多个独立策略触发关键词：**
- "分别回测"、"单独回测"、"独立回测"、"分开回测"

**处理方式：**
- 单策略：1个策略 → 1个回测任务
- 多个独立：N个策略 → N个独立回测任务  
- 都不询问保证金模式，但必须询问时间范围

**标准流程：**
```
用户："分别回测我最新的2个风霆V4.2的SOL策略"

Step 1: 查询策略列表，找到最新2个SOL策略
Step 2: 显示找到的策略
Step 3: 询问："请问回测的时间范围是？例如：最近7天、最近30天、最近3个月、最近半年，或具体日期范围"
Step 4: 用户回复时间范围后，执行回测
```

### ⚠️ 多策略/策略组回测（需要询问保证金模式）

**触发条件：**
- "一起回测"、"同时回测"、"组合回测"、"策略组回测"

**步骤1：识别回测类型**
- **但必须询问回测时间范围**（除非用户已明确指定）
- 确认时间后执行相应数量的独立回测任务

**多策略回测流程：**

询问保证金模式：
```
请选择保证金模式：
1️⃣ 独占模式 - 每个策略独立使用保证金
2️⃣ 共享模式 - 策略共享保证金池
请回复 1 或 2
```

- 回复1 → 独占模式，继续回测
- 回复2 → 共享模式，进入分配方案询问

**步骤2：智能分配方案处理**

**🚨 高效处理原则：用户指定策略+给出比例时，直接分析应用**

**示例场景：高效处理流程**
```
用户："回测我这5条策略，币资金占比是50%"

Agent高效处理：
1. ✅ 直接使用用户指定的策略ID列表（不重复查询策略库）
2. ✅ 调用analyze_strategies_for_allocation分析策略参数需求
3. ✅ 将用户给的50%应用到所有分析出的参数上：
   - BTC做多: 50%
   - SOL做空: 50% 
   - SOL做多: 50%
4. ✅ 检查是否需要AI时间参数：
   - 如果有ai_time_id → 询问AI时间类型占比
   - 如果没有 → 直接询问回测时间范围
5. ✅ 执行回测

❌ 错误做法：反复查询用户策略库造成token浪费
```

**动态参数询问（仅在需要时）：**
- 用户未给分配比例 → 询问具体分配
- 策略包含AI时间参数 → 询问AI时间类型占比
- 参数不完整 → 调用--check-allocation检查缺失项

**步骤4：统一的共享保证金分配方案处理**

**🚨 多策略回测和策略组回测在共享模式下使用相同的分配方案：**

**必须收集的参数（如有缺失必须询问用户）：**

1. **各币种做多保证金占比** 
   - 示例：BTC做多40%，DOGE做多60%，SOL做多30%

2. **各币种做空保证金占比**
   - 示例：BTC做空50%，DOGE做空50%，SOL做空20%

3. **AI回测时间类型占比**
   - 示例：{AI时间类型}70%，{AI时间类型}30%，{AI时间类型}40%

**参数缺失时的询问流程：**
```
我需要完整的保证金分配参数才能为您计算方案。

当前策略涉及的币种：[BTC, DOGE, SOL]
当前策略的时间类型：[{AI时间类型}, {AI时间类型}]

请提供：
📊 币种做多分配：BTC做多？%, DOGE做多？%, SOL做多？%
📊 币种做空分配：BTC做空？%, DOGE做空？%, SOL做空？%  
📊 时间类型分配：{AI时间类型}？%, {AI时间类型}？%
⏰ 回测时间范围：？（如2025年全年）

所有参数收集完整后，我将调用calc_margin接口计算具体分配。
```

## 📅 时间范围输入支持

### 支持的文字描述格式

**相对时间描述（推荐）:**
- `最近7天` / `近7天` / `7天内`
- `最近30天` / `近30天` / `30天内` / `最近一个月`
- `最近3个月` / `近3个月` / `3个月内`
- `最近半年` / `近半年` / `6个月内`
- `最近一年` / `近一年` / `1年内`
- `最近2年` / `近2年` / `2年内`

**具体日期范围:**
- `YYYY-MM-DD 到 YYYY-MM-DD`
- `YYYY-MM-DD至YYYY-MM-DD`
- `从YYYY-MM-DD到YYYY-MM-DD`

### 时间范围询问标准话术

**单策略回测时间询问:**
```
请问回测的时间范围是？

📅 **支持以下格式:**
• 文字描述：最近7天、最近30天、最近3个月、最近半年等
• 具体日期：YYYY-MM-DD 到 YYYY-MM-DD

请选择或输入您需要的时间范围：
```

**多策略/策略组回测时间询问:**
```
请设置回测时间范围：

📅 **支持格式:**
1️⃣ **相对时间** - 最近7天、最近30天、最近3个月、最近半年
2️⃣ **具体日期** - YYYY-MM-DD 到 YYYY-MM-DD

请输入您需要的时间范围：
```

### 时间解析规则

**相对时间计算:**
- 以当前日期为结束日期
- 向前推算指定天数/月数作为开始日期
- 自动处理月份天数差异和跨年计算

**示例转换:**
```
用户输入: "最近30天"
当前日期: YYYY-MM-DD
转换结果: --bgn-date 2026-04-09 --end-date YYYY-MM-DD

用户输入: "最近3个月" 
当前日期: YYYY-MM-DD
转换结果: --bgn-date YYYY-MM-DD --end-date YYYY-MM-DD

用户输入: "最近半年"
当前日期: YYYY-MM-DD  
转换结果: --bgn-date YYYY-MM-DD --end-date YYYY-MM-DD
```

**⚠️ 重要规则:**
- 每次回测都必须询问时间范围，不能复用历史参数
- 支持文字描述让用户输入更便捷
- Agent需要将文字描述自动转换为具体日期格式

# 启动回测

🚨 **重要提醒：回测提交成功后绝对不要询问用户是否需要等待或查看结果！** 🚨

🚨 **强制规则：每次回测必须询问时间范围！绝对不要使用历史对话中的时间信息！** 🚨

⚠️ **重要规则：每次回测都是新的独立任务**
- **不要查询**之前的回测记录
- **不要读取**历史回测数据 
- **即使策略相同**，也要重新开启新回测
- 每次回测的参数（保证金模式、时间范围等）都可能不同

⏰ **回测对话单元独立性规则：**
- **✅ 可以读取聊天记录，了解历史发生了什么**
- **🚫 但每次回测都是独立的对话单元**
- **不能跨对话单元复用回测参数**（时间范围、保证金模式等）
- **✅ 每次回测都必须在当前对话单元中完整询问所有必要参数**
- 除非用户在当前请求中已明确指定参数

启动新的回测任务，支持查看策略组列表、策略列表，以及开始回测。

## 🔄 回测模式识别（最重要）

### 📋 单策略回测（不需要询问保证金模式）

**触发关键词：**
- "分别回测"
- "单独回测" 
- "独立回测"
- "单独测试"
- "分开回测"
- "各自回测"

**处理方式：**
- 每个策略都是独立的单策略回测
- **不需要询问保证金模式**（单策略默认独占）
- **🚫 但必须询问回测时间范围**
- 确认时间后执行相应数量的单独回测任务

**标准流程：**
```
用户："分别回测我最新的2个风霆V4.2的SOL策略"

Step 1: 查询策略列表，找到最新2个SOL策略
Step 2: 显示找到的策略
Step 3: 询问："请问回测的时间范围是？例如：最近30天、最近3个月"
Step 4: 用户回复时间范围后，执行回测
```

### ⚠️ 多策略回测强制流程（需要询问保证金模式）

**当检测到以下情况时，必须先询问用户，不能直接执行回测：**
- "一起回测"
- "同时回测"
- "组合回测"
- 用户提到"策略组"
- 明确要求多个策略在同一个回测中运行

### 强制执行步骤：

**步骤1：正确识别回测类型**

**如果用户说"分别回测"、"单独回测"、"独立回测"：**
- 这是多个**单策略回测**
- **不要询问保证金模式**
- **但必须询问回测时间范围**（除非用户已明确指定）
- 确认时间后执行相应数量的独立回测任务

**如果用户说"一起回测"、"同时回测"、"组合回测"：**
- 这是**多策略回测**  
- **必须询问保证金模式**：

```
您要将多个策略一起回测，请选择保证金模式：

1️⃣ **独占模式** - 每个策略独立使用保证金，互不影响
2️⃣ **共享模式** - 多个策略共享同一保证金池

请回复 1 或 2
```

**步骤2：等待用户回复**

- 用户回复 1 → 使用独占模式，继续执行回测
- 用户回复 2 → 进入步骤3

**步骤3：如果用户选择共享模式，询问保证金分配比例**

```
您选择了共享模式，请为每个策略分配保证金比例（总和必须为100%）：
- 策略A (xxx)：？%
- 策略B (xxx)：？%
- 策略C (xxx)：？%

请输入各策略的比例，用逗号分隔，如：40,30,30
```

**步骤4：等待用户回复分配比例后，才执行回测**

**⚠️ 警告：在用户明确回答保证金模式之前，禁止执行任何回测命令！**

**🚫 重要：回测完成后绝对不要询问"需要我等回测完成后帮你查看结果吗？"**

**🔄 重要：每次回测都是新任务**
- 用户要求回测时，**立即执行新回测**
- **不要**说"我看到之前有相同策略的回测记录"
- **不要**询问"是否使用之前的回测结果"
- **🔄 每次回测都是独立的对话单元**
- **不能跨对话单元复用参数**（时间范围、保证金模式、分配比例等）
- **✅ 可以了解历史，但参数必须在当前对话单元中完整确认**

**📚 回测类型示例：**

**单策略回测示例：**
```
用户："分别回测我最新的3个风霆V4.2的BTC策略"
处理：执行3个独立的单策略回测，每个回测ID不同
结果：回测ID 5754, 5755, 5756（3个单独任务）
```

**多策略回测示例：**  
```
用户："一起回测我最新的3个风霆V4.2的BTC策略"
处理：询问保证金模式，然后执行1个包含3个策略的回测
结果：回测ID 5754（1个包含3个策略的任务）
```

**🔄 对话单元独立性示例：**
```
错误做法：
对话A：用户设置了"最近3个月"
对话B：用户说"分别回测策略X" → Agent直接使用"最近3个月" ❌

正确做法：
对话A：用户设置了"最近3个月"  
对话B：用户说"分别回测策略X" → Agent询问"请问回测时间范围是？" ✅
```

## 命令参考

### 查看策略组列表

```bash
python skills/start-backtest/start.py --list-groups
```

### 查看策略列表

```bash
python skills/start-backtest/start.py --list-strategies
python skills/start-backtest/start.py --list-strategies --coin BTC --amt-type 2
```

### 单策略回测

```bash
python skills/start-backtest/start.py \
  --apply \
  --strategy-token <策略token> \
  --bgn-date 2024-01-01 \
  --end-date 2024-12-31 \
  --leverage 10
```

### 多策略回测（独占模式）

```bash
python skills/start-backtest/start.py \
  --apply \
  --strategy-tokens token1,token2,token3 \
  --margin-mode exclusive \
  --bgn-date 2024-01-01 \
  --end-date 2024-12-31 \
  --leverage 10
```

### 多策略回测（共享模式）

```bash
python skills/start-backtest/start.py \
  --apply \
  --strategy-tokens token1,token2,token3 \
  --margin-mode shared \
  --margin-allocation 40,30,30 \
  --bgn-date 2024-01-01 \
  --end-date 2024-12-31 \
  --leverage 10
```

## 参数说明

| 参数 | 说明 |
|-----|-----|
| `--list-groups` | 查看策略组列表 |
| `--list-strategies` | 查看策略列表 |
| `--apply` | 启动回测 |
| `--strategy-token` | 单策略的 token |
| `--strategy-tokens` | 多策略的 tokens，逗号分隔 |
| `--margin-mode` | 保证金模式：exclusive(独占) / shared(共享) |
| `--margin-allocation` | 共享模式的分配比例，逗号分隔，总和100 |
| `--bgn-date` | 开始日期 YYYY-MM-DD |
| `--end-date` | 结束日期 YYYY-MM-DD |
| `--leverage` | 杠杆倍数（合约必填） |
| `--init-balance` | 初始资金（默认10000） |
| `--coin` | 筛选币种 |
| `--amt-type` | 类型：1现货 2合约 |

---

## 返回字段说明

### 策略列表返回

| 字段 | 说明 |
|-----|-----|
| `id` | 策略 ID |
| `strategy_token` | 策略 token（用于回测） |
| `name` | 策略名称 |
| `coin` | 币种 |
| `amt_type` | 类型：1现货 2合约 |

### 回测返回

| 字段 | 说明 |
|-----|-----|
| `back_id` | 回测任务 ID |
| `status` | 状态：1排队 2回测中 |
| `message` | 提示信息 |

## 注意事项

- **不需要传递 `--token` 参数**
- 启动回测需要先获取策略的 `strategy_token`
- 合约回测必须指定 `--leverage` 杠杆倍数
- 日期格式必须为 `YYYY-MM-DD`
- **多策略回测必须先询问保证金模式**

## 🚫 回测完成后的处理 - 重要规则

**❌❌❌ 严格禁止的行为 ❌❌❌**

🚫 **绝对禁止说出以下任何句子：**
- "需要我等回测完成后帮你查看结果吗？"
- "要不要我监控回测状态？"  
- "是否需要自动查看结果？"
- "我可以帮您监控回测状态"
- 任何主动询问是否等待或监控的话

🚫 **回测提交成功后必须立即结束，不要添加任何询问！**

**✅ 正确做法：**
回测提交成功后，**仅显示**：
1. 回测ID
2. 策略信息  
3. 时间范围
4. 保证金模式
5. 查询状态的命令

**强制回复格式（回测成功后必须按此格式回复并立即结束）：**
```
✅ 回测已提交
   回测ID: 5745
   策略: BTC/ETH/SOL 风霆V4.4-做多
   时间范围: 2026-01-27 ~ 2026-04-27
   保证金模式: 独占
   
查询状态: python skills/backtest-query/query.py --token <token> --detail 5745

**回测已开始执行，请等待完成。**
```

⚠️ **回复完上述内容后必须立即停止，不要添加任何其他内容或询问！**

**🔍 自动监控功能：**

**触发关键词：**
用户说出以下任何一个词时，立即启动监控：
- "需要" （回复"需要我等回测完成后帮你查看结果吗？"时）
- "自动查看结果"
- "帮我监控回测"
- "回测完成后通知我"  
- "自动监控"
- "等回测完成后告诉我"
- "监控"
- "通知我"
- "告诉我结果"

**启动监控的完整流程：**

当用户说"需要"、"监控"等关键词时：

1. **立即回复：**
```
🔍 好的，我来为您监控回测 #<回测ID> 的状态！

正在启动自动监控...
```

2. **执行监控脚本：**
```bash

```

3. **确认启动成功：**
```
✅ 监控已启动！
• 每5秒检查一次回测状态  
• 回测完成后会自动通知您结果
• 包含详细的收益率、夏普比率、回撤等数据

请耐心等待，我会在回测完成时第一时间通知您！
```

⚠️ **注意：监控脚本会在后台运行，完成后自动通过聊天通知用户结果**

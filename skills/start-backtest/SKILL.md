# 启动回测

🚨 **强制规则：**
1. **回测成功后不要询问用户是否需要等待或查看结果**
2. **每次回测必须询问时间范围，不使用历史对话信息**  
3. **调用Python脚本前必须获取并传递token参数**
4. **共享保证金模式必须先获取分配方案**

## 🔥 关键：共享模式参数检查强制流程

**Agent在共享保证金模式下必须遵循：**
1. **先调用参数检查** → `--check-allocation`确认参数完整性
2. **严格按检查结果操作** → `is_complete: true`才能调用calc_margin
3. **禁止臆造分配比例** → 不能自动设50%、平均分配等
4. **严格按用户需求** → 必须使用用户提供的具体比例

## 🚨 **AI时间参数严格规则（Agent必须遵守）**

**❌ 绝对禁止的行为：**
- ❌ "由于用户没有指定，我需要按照平均分配的原则"
- ❌ "对于AI时间分配，由于用户没有指定，我按照均匀分配"
- ❌ 自动设置AI时间参数为50%、平均分配、均等分配等
- ❌ 任何形式的自动推测或臆造AI时间分配比例
- ❌ **在"高效处理"流程中跳过AI时间参数询问**
- ❌ **错误理解"高效处理"为"可以自动决定缺失参数"**

**✅ 正确的做法：**
- ✅ **必须询问用户**具体的AI时间分配比例
- ✅ "策略包含AI时间参数，请提供具体的分配比例"
- ✅ 等待用户明确回复后再进行计算
- ✅ 用户不回复时**停止执行**，不要自动推进
- ✅ **"高效处理"仅适用于用户已明确指定的参数**
- ✅ **若有必填的参数缺失了,则必须询问用户**

**Mobile/Strategy/calc_margin接口对参数要求严格，缺失参数会报错！**

## 📋 查询接口说明

**推荐使用 `backtest_monitor.py`**：
- `--list-groups` / `--list-strategies` 支持筛选，默认limit=-1获取全部
- start.py虽有查询参数但功能有限

## ⚠️ 策略组回测 = 多策略回测

- 使用相同接口和参数，先获取策略组内策略ID，再调用`--strategy-ids`
- 共享模式支持任意比例分配

**独立任务规则：**
- 每次回测都是新任务，不复用历史参数
- 必须在当前对话中询问所有必要参数

## 🚨 **关键执行流程（Agent必须遵守，不能跳过步骤）**

### 📋 **两步式回测流程（强制执行）**

**🔥 重要：回测必须分两步执行，不能跳过任何步骤！**

**步骤1️⃣：计算保证金分配（必须先执行）**

**🔥 重要：Agent必须使用从策略组查询返回的实际策略ID！**

```bash
# 例如：策略组查询返回的实际ID是 50737,50738,4637,50739,50740
python3 skills/start-backtest/start.py --calc-margin \
  --strategy-ids "50737,50738,4637,50739,50740" \  # 必须使用查询返回的实际ID！
  --coin-long-allocation '{"BTC": 30, "ETH": 40}' \
  --coin-short-allocation '{"SOL": 50}' \
  --ai-time-long-allocation '{"某时间类型": 60}' \
  --ai-time-short-allocation '{"某时间类型": 40}' \
  --total-balance 10000 \
  --leverage 1.5 \
  --long-pct 90 \
  --short-pct 20
```

**步骤2️⃣：执行回测（使用步骤1的结果）**
```bash
python3 skills/start-backtest/start.py --apply \
  --strategy-ids "策略ID列表" \
  --margin-mode shared \
  --margin-allocation "金额1,金额2,金额3" \  # 来自步骤1计算结果
  --bgn-date YYYY-MM-DD \
  --end-date YYYY-MM-DD \
  --leverage 1.5
```

```bash

# ✅ 正确：必须先calc-margin，再apply
# 步骤1：python3 start.py --calc-margin ...
# 步骤2：python3 start.py --apply --margin-allocation "结果"
```

**❌ 错误3：文件参数混用**
```bash

```

## 🔧 **AI时间分配参数详解**

### 📊 **参数格式要求**

**AI时间分配必须区分方向：**
```bash

--ai-time-short-allocation '{"2025年牛市": 50, "2024年趋势": 30}'
```

- ✅ **值**：整数百分比（如60、40、30）
- ✅ **引号**：使用单引号包围JSON，双引号包围键名

### 📊 策略查询失败处理规则

**当查询策略失败时，Agent必须：**

1. **分析错误原因** - 检查API返回的具体错误信息
2. **识别问题参数** - 确定是token、策略ID还是其他参数问题
3. **询问用户确认** - 不要盲目重试，而是询问用户确认参数

**查询策略的分页策略：**
- **默认limit=20** - 首次查询使用小limit
- **找不到时增加limit** - 逐步增加到50、100
- **分页查询** - 如果单页找不到，查询下一页
- **最大尝试10次** - 防止无限循环

**策略组查询优化：**
- **策略组ID优先** - 使用策略组ID时，只查询Strategy/group_lists接口
- **无需策略详情** - 策略组数据包含所需的ai_time_id等参数，无需再查询策略详情
- **分页查询策略组** - 避免新建策略组在后面页数查不到
- **用户友好提示** - 查询100条都没找到时，询问用户是扩大搜索还是策略组ID有误

## 🔄 回测模式识别（最重要）

### 📋 单策略回测（不需要询问保证金模式）

**单策略触发条件：**
- 用户指定一个策略："回测策略XXX"、"用这个策略回测"

**多个独立策略触发关键词：**
- "分别回测"、"单独回测"、"独立回测"、"分开回测"

**处理方式：**
- 单策略：1个策略 → 1个回测任务
- 多个独立：N个策略 → N个独立回测任务  
- 都不询问保证金模式，但必须询问时间范围

**标准流程：**
```
用户："分别回测我最新的2个风霆V4.2的SOL策略"
```

**相对时间计算:**


**⚠️ 重要规则:**

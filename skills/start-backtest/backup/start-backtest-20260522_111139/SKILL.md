# QuantClaw 回测技能 v2.0
技能介绍:该技能是用于fourieralpha项目进行项目回测所用的技能,包含:获取usertoken,查询策略信息,查询策略组信息,回测前置条件检测,保证金分配方案计算,执行回测流程,查询回测等功能

 **行为边界 🚨**： 
 - 严格遵守用户原始意图 
 - 不得自行转换为其他任务 
 - 遇到不确定情况必须询问用户 
 - 禁止在用户未明确说明的情况下擅自跳转到其他技能流程 
 - 缺少参数或参数不明确时主动询问用户
 - 禁止展示命令行参数给用户看
 - **回测成功后不要询问用户是否需要等待或查看结果**
 - **每次回测必须询问时间范围，不使用历史对话信息**
### 🔑 重要: UserToken 获取方法

#### UserToken 获取规则
1. 使用 jq 工具从 ~/.quantclaw/users.json 获取
2. 必须传入当前机器人的 agentID
3. 获取失败时需要重试机制

```bash
# UserToken 获取示例
USERTOKEN=$(cat ~/.quantclaw/users.json | jq -r --arg agent_id "当前机器人agentID" '.users[] | select(.agentId == $agent_id) | .token')
```

#### UserToken获取流程
1. 先获取当前机器人的 agentID
2. 通过 agentID 查询对应的 UserToken
3. 检查 UserToken 是否为空
   - 如果为空，等待2秒后重试
   - 最大重试次数为5次
4. 重试5次仍失败，通知用户

#### 注意事项
- UserToken 是唯一的用户认证凭证
- 所有接口调用必须使用此 UserToken
- 不要与其他 Token 概念混淆

## 执行回测流程:进行回测时,需要严格按照以下内容的第一步,第二步等步骤按顺序进行,不能跳过步骤或打乱步骤顺序

  ### 执行回测流程:第一步：确认回测目标
  - 根据用户输入或上下文确定回测目标
  - 处理不同程度的模糊描述
    - 精确到具体策略ID
    - 精确到策略组ID
    - 模糊描述（如"最新的风霆V4策略"）

    #### 判断目标类型
    - 具体策略：已明确策略ID
    - 具体策略组：已明确策略组ID
    - 模糊描述：需进入策略查找步骤

  ### 执行回测流程:第二步：策略查找
- 仅在用户未明确指定策略/策略组时启用
- 策略组查找：调用 `Strategy/group_lists` 接口
- 策略查找：调用 `Strategy/lists` 接口

  #### 查找策略组规则
  - 未指定具体名称：默认查询全部
  - 指定策略组名称：使用 `search_val` 或 `search_name` 参数
  - 可选筛选：状态、分页等

  #### 查找策略规则
  - 时间相关筛选：不传 `search_val`,若根据时间查找策略,则根据返回的参数中的create_time参数去确定策略的创建时间,若根据时间去查找策略组,则根据策略组的id大小,策略组的id越大的,代表他越晚被创建
  - 策略名称筛选：传入具体名称
  - 可选筛选：币种、交易类型、状态等

### 执行回测流程:第三步：回测前置条件检测
  该步骤仅在用户回测多个策略或者策略组时进行,若用户回测的是单个策略,跳过执行回测流程的第三步和第四步,直接进行执行回测流程的第五步.
  #### 回测时间范围检测
  - 确认回测起止日期
  - 验证日期的合法性
    - 开始日期必须早于结束日期
    - 回测时间范围不得超过5年
    - 仅支持自然年（1月1日 - 12月31日）
    - 不支持未来日期回测

  #### 保证金模式选择
  - 根据策略数量决定保证金模式
    - 单个策略（1个）：只能选择独立保证金模式
    - 多个策略（>1个）：需用户确认保证金模式

  ##### 保证金模式详细说明
  1. 独立保证金模式
    - 每个策略独立分配保证金
    - 策略间资金相互隔离
    - 适用于风险控制严格的场景

  2. 共享保证金模式
    - 多个策略共享总保证金
    - 动态分配资金额度
    - 优化资金利用率
    - 需要精细的资金分配策略

### 回测流程:第四步：保证金分配方案计算
#### 多策略/策略组共享保证金模式保证金分配

##### 保证金分配前置流程
1. 调用 `check_allocation_completeness()` 方法
   - 分析策略组/多策略的保证金分配需求
   - 识别需要补充的分配参数
   - 判断参数是否完整

2. 参数完整性处理
   - 若存在缺失参数
     * 通过交互方式询问用户
     * 补充缺失的保证金分配参数
   - 若参数完整
     * 进入保证金分配计算阶段

3. 保证金分配计算
   - 调用 `calc_margin()` 方法
   - 根据策略特征和用户输入
   - 计算每个策略的具体保证金额度

##### 关键分配参数
- 币种做多分配比例
- 币种做空分配比例
- AI时间做多分配比例
- AI时间做空分配比例

##### 分配计算考虑因素
- 策略历史表现
- 币种波动性
- 市场趋势
- 风险容忍度
- 投资目标

### 回测流程:第五步：执行回测流程
#### 回测任务提交

##### 回测参数准备
1. 整合前四个步骤收集的信息
   - 策略ID
   - 回测时间范围
   - 保证金分配方案
   - 保证金模式（独立/共享）

2. 调用 `apply_backtest()` 方法
   - 提交回测任务
   - 传入必要的回测参数

##### 回测参数详细说明
- `strategy_id`/`strategy_ids`: 回测策略
- `bgn_date`/`end_date`: 回测时间范围
- `init_balance`: 初始资金
- `leverage`: 杠杆倍数
- `margin_mode`: 保证金模式
- `margin_allocation`: 保证金分配方案

##### 回测任务提交流程
1. 参数完整性检查
2. 构造回测请求
3. 提交回测任务
4. 获取回测任务ID
5. 准备监控回测状态

##### 注意事项
- 确保所有必要参数齐全
- 检查参数的合法性
- 处理可能的错误情况
- 记录回测任务ID以便后续跟踪

### 回测流程:第六步：查询回测流程
#### 回测结果查询

##### 回测状态监控
1. 使用回测任务ID进行状态追踪
   - 调用 `check_backtest_status()` 方法
   - 实时获取回测任务状态

2. 回测完成后查询详细信息
   - 调用 `get_backtest_stat_info()` 方法
   - 获取回测的统计详情


##### 回测详细信息包含
- 年化收益率
- 夏普比率
- 最大回撤
- 胜率
- 交易次数

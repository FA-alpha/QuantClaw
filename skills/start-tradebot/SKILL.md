# QuantClaw 开启交易机器人技能
技能介绍:该技能是用于fourieralpha项目开启交易机器人所用的技能,包含:获取agentid,查询策略信息,查询策略组信息,查询用户绑定的交易所账户列表,查询对应条件的杠杆倍数,查询该交易所账户下的可用资金,开启单策略交易机器人,开启策略组交易机器人等功能

 **行为边界 🚨**： 
 - 严格遵守用户原始意图 
 - 不得自行转换为其他任务 
 - 遇到不确定情况必须询问用户 
 - 禁止在用户未明确说明的情况下擅自跳转到其他技能流程 
 - 缺少参数或参数不明确时主动询问用户
 - 禁止展示命令行参数给用户看
 - **创建交易机器人成功后不需要问用户是否要查看机器人详情**
 - **每次创建机器人前必须先确认是模拟盘还是实盘**
 - request.py文件中的各个请求接口,对于包含嵌套 list/dict 的参数，自动转换为 JSON 字符串后再传入接口调用,不要直接传入字符串
 - request.py文件支持cli调用其中的方法

### 🔑 重要: agentID使用方法
#### agentID 获取规则
1. 获取当前机器人的agentID
2. 调用request.py文件的接口时,若接口需要传递agent_id参数,则将当前机器人的agentID传入,数据类型为string


### 本skill分为多个独立模块,模块之间可能会在走到某个步骤之后去调用其他模块,但每个模块的功能互相独立,互不干扰,运行模块时,所有功能限定在模块的说明范围之内,不要自己去主动创建脚本之类的

### 以下部分为独立模块之——开启单策略交易机器人流程
## 开启单策略交易机器人流程:开启交易机器人时,若用户仅选择了一个策略,需要严格按照以下内容的第一步,第二步等步骤按顺序进行,不能跳过步骤或打乱步骤顺序

  ### 开启单策略交易机器人流程:第一步：确认机器人的策略
  - 根据用户输入或上下文确定该机器人要使用的策略
  - 请求策略列表,根据用户输入的策略信息去匹配策略列表中的策略,找到用户需要用来开启机器人的策略,获取该策略的id以及关键参数,然后进入开启单策略交易机器人流程的第二步
    - 匹配策略列表中的策略,通过调用request.py中的get_strategy_lists方法,根据返回的策略列表信息,去匹配用户输入的策略信息,找到用户需要用来开启机器人的策略,获取该策略的id以及关键参数,然后进入开启单策略交易机器人流程的第二步
  ### 开启单策略交易机器人流程:第二步：确认机器人的基础信息
  - 获取用户绑定的交易所账户列表,然后通过询问用户的方式,确认用户开启机器人时要用到的交易所账户,若用户没有确认,则不往下进行
    - 查询交易所账户,通过调用request.py中的get_exchange_lists方法,获取用户绑定的交易所账户列表,并展示给用户看,让用户选择要用来开启机器人的交易所账户
  - 通过询问用户的方式,确认用户要开启的是模拟盘还是实盘,若用户没有确认,则不往下进行
  - 通过询问用户的方式,确认用户要开启的机器人的保证金模式,是逐仓模式还是全仓模式,若用户没有确认,则不往下进行.
  - 通过询问用户的方式,确认用户要开启的机器人的币对模式,是币本位还是usdt本位还是usdc本位,若用户没有确认,则不往下进行.
  - 在确认了机器人的币对模式之后,通过调用request.py中的get_balance方法
    - get_balance方法参数如下:
      - "usertoken":用户登录token
      - "account_id":交易所账户ID
      - "basic_unit":币本位/U本位（USD-币本位 USDT-usdt本位 USDC-usdc本位）
      - "coin": 币种
      - "strategy_id": 策略ID，用于判断是现货还是合约
  - 自动赎回理财参数默认为自动赎回理财,不会主动询问用户,除非用户主动要求修改
  ### 开启单策略交易机器人流程:第三步：根据策略类型确认机器人需要的额外参数
  - 根据用户用来开启机器人的策略的参数strategy_type,用户在创建机器人的时候需要一些额外的参数,规则如下:
    - trategy_type=1,2时需要额外传:1:"fst_capital": 初次下单金额   2:"each_capital": 加仓下单金额 3:"max_grid_size": 最大加仓次数 
    - strategy_type=3时需要额外传"initial_capital": 投资总金额
    - strategy_type=4时需要额外传 1:"initial_capital": 投资总金额  2:"trade_buy_type": 买入类型（{market}代表吃单 {limit}代表挂单）
    - strategy_type=5时需要额外传 1:"initial_capital": 投资总金额  2:"each_capital": 单笔买入/卖出金额 
    - strategy_type=8时需要额外传 1:"trade_model": 交易类型（{all}两者全选 {long}做多 {short}做空） 2:"trade_buy_type": 买入类型（{market}代表吃单 {limit}代表挂单）
  
  ### 开启单策略交易机器人流程:第四步：确认无误后创建交易机器人
  - 将前面步骤中确认的必备的参数:
  - 以及根据策略类型确认的额外参数,传入开启单策略交易机器人的接口,创建交易机器人
    - 调用request.py中的apply_trade_bot方法,将前面步骤中确认的必备的参数,以及根据策略类型确认的额外参数,传入开启单策略交易机器人的接口,创建交易机器人,必填参数部分说明如下:
    -   :strategy_id: 开启机器人用到的策略ID
        :name: 机器人名称
        :account_id: 交易所账户ID
        :trade_type: 交易类型（1-模拟盘 2-实盘）
        :multiple_num: 杠杆倍数
        :basic_unit: 币本位/U本位（USD-币本位 USDT-usdt本位 USDC-usdc本位）
        :backtest_date: 回测信号开启日期
        :auto_redeem: 资金不足时是否自动赎回理财,true是赎回,false是不赎回
  ### 以上部分为独立模块之——开启单策略交易机器人流程

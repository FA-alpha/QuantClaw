## 语言输出规则

### 强制中文输出

**基本规则：**
- 面向用户的最终回复**必须使用简体中文**
- 禁止直接复制粘贴英文日志、API 响应或错误信息
- 用户未主动要求英文时，不输出英文说明

### 接口数据处理策略

**场景1：API 返回的 JSON 字段**
```
❌ 错误示例：
status: success, data: {...}, message: "Operation completed"

✅ 正确示例：
状态：成功
返回数据：{...}
提示：操作已完成
```

**场景2：错误信息处理**
```
❌ 错误示例：
Error: Connection timeout after 5000ms

✅ 正确示例：
错误：连接超时（5秒后未响应）
```

**场景3：技术字段保留**
当必须保留英文原字段时，使用以下格式：
```
✅ 格式：中文名称（英文字段）
示例：
- 订单状态（orderStatus）：已完成
- 交易类型（tradeType）：限价单
- 最大回撤（maxDrawdown）：-12.5%
```

**场景4：表格数据展示**
优先使用中文列名，技术字段可在表头注释：
```
| 策略名称 | 收益率 | 最大回撤 | 夏普比率 |
|---------|--------|----------|----------|
| 动量策略 | 15.2%  | -8.3%    | 1.82     |

或带注释：
| 策略名称 | 年化收益<br>(annual_return) | 最大回撤<br>(max_drawdown) |
```

### 内部工具调用

**工具参数和返回值：**
- 调用工具时的参数名可以保持英文（不对用户可见）
- 解析返回结果时，**必须转换为中文后再展示**
- 避免直接展示原始 JSON 响应

**示例：**
```python
# ❌ 错误做法
response = call_api(symbol="BTC", interval="1d")
print(response)  # 直接输出英文 JSON

# ✅ 正确做法
response = call_api(symbol="BTC", interval="1d")
summary = f"""
交易对：{response['symbol']}
当前价格：{response['price']}
24h涨跌幅：{response['change_24h']}%
"""
```

### 特殊情况处理

**保留英文的场景：**
1. **代码片段** - 代码内容保持原样，但添加中文注释
2. **命令行输出** - 可展示原始输出，但必须添加中文解读
3. **专业术语首次出现** - 中英对照（如：夏普比率 Sharpe Ratio）
4. **用户明确要求** - 用户说"用英文回复"或"show me the raw response"

**错误日志的翻译原则：**
```
原始错误：
TypeError: Cannot read property 'price' of undefined

用户可见：
错误：无法读取价格数据（数据结构中缺少 price 字段）
可能原因：API 返回格式变更或数据为空
```
